宿舍管理系统项目的后端部分是我自己写的，借用AI更改小bug以及提供问题思路；

前端部分本来我写的挺简单的，但是bug太多加上不熟练，用AI大改了特别多，因此前端的代码量也特别多。

## 卡住的地方

**1. token 怎么处理？**

每次发请求都要手动取 token 加 header，又麻烦又容易漏。知道有拦截器但不会用。

**2. 表格筛选怎么做？**

想加个下拉框筛选报修状态，用 v-if 写出来数据不对，调试半天没搞明白。

**3. 路由守卫不会写**

想实现“没登录自动跳回登录页”，照着教程写 router.beforeEach 一直报错。

------

## AI 帮忙解决的问题

**1. token 自动携带**：AI 给了 axios 拦截器代码，复制进去就能用了

**2. 表格筛选**：AI 用 computed 重写，加了下拉框联动

**3. 路由守卫**：AI 给了完整代码，还顺便加了 pinia 存用户信息

------

## AI 帮我解决的后端问题

**1. 学生和管理员共用 token**

在 Filter 里加 userType 判断，学生不能访问 /admin，管理员不能访问 /student

**2. 图片无法显示**

配置静态资源映射 WebConfig，让 /uploads/** 指向本地文件夹

------

## 我遇到的坑

| 坑                 | 原因                                       | 解决                        |
| :----------------- | :----------------------------------------- | :-------------------------- |
| CORS 跨域          | Filter 拦截了 OPTIONS 预检请求             | 放行 OPTIONS，直接 return   |
| 类型转换错误       | Filter 存 Long，Controller 用 Integer 接收 | 统一用 Integer                |
| 重复学号报空指针   | 查询返回 null 直接调 .intValue()           | 先判空或用 COUNT(*)         |
| 图片访问 404       | 没配置静态资源映射                         | 加 WebConfig 映射           |
| 管理员列表字段空白 | 返回下划线格式，前端要驼峰                 | SQL 加别名                  |
| 学生能进管理员页面 | 权限判断写反了                             | 改成 `userType != 1` 时拦截 |

------

## 我的调整

虽然前端 AI 加的代码多，但我也不是完全不管：

- **校验规则**：学号工号的正则是自己写的

- **功能精简**：删掉了一些不必要的弹窗

- **逻辑理解**：看不懂的地方再问 AI，或在控制台打印调试

  

## 心得

1. 前后端联调时，先用 Apifox 测后端，确认没问题再看前端
2. 统一类型很重要，我被 Long 和Integer整累了
3. CORS 问题要在 Filter 最前面处理 OPTIONS
4. SQL 别名能避免字段映射问题
5. 关键位置加 `log.info`
